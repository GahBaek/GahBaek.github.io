메모리 보호 기법

2026·04 Note
  • 메모리 보호 기법

- checksec

실행 파일에서 제공하는 보안 메커니즘을 알기 쉬운 형태로 표시해준다.


gef➤  checksec
[+] checksec for '/home/hyeoni/Documents/Dreamhack/0410/r2s'
Canary                        : ✓
NX                            : ✘
PIE                           : ✓
Fortify                       : ✘
RelRO                         : Ful시
  • NX - No Execute

- 코드 실행 가능한 메모리 영역과 쓰기 가능한 메모리 영역을 분이라는 보호 기법.

  • PIE - Position Independent Execution

- PIE 보호 기법이 적용되면 실행할 때마다 바이너리의 주소가 랜덤화된다.

  • RelRO - RELocation Read Only

- 데이터가 배치된 메모리의 어느 부분에 읽기 전용 속성을 부여하는 기법이다. → 즉, 메모리에 배치되어 있는 데이터를 변경하지 못하도록 보호한다.

- Full RelRO / Partial RelRO / No RelRO

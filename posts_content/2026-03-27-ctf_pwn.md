## PWNTOOL

pwntools is a CTF framework and exploit development library.

### 주요 기능

- **`process`**
    - 로컬 바이너리를 대상으로 exploit 을 할 때 사용
    - test로 사용
- **`remote`**
    - 원격 서버를 대상으로 exploit을 할 때 사용
    - 실제 실행 시 사용

send

- **`p.send(data)`**
    - 데이터 입력
- **`p.sendline(data)`**
    - 데이터에 ‘\n’ 추가해서 입력
- **`p.sendafter('output', data)`**
    - output 내용이 출력되면 data 입력
- **`p.sendlineafter('output', data)`**
    - output 내용이 출력된다면 data + ‘\n’을 입력

recv

- **`p.recv(n)`**
    - n byte 만큼 데이터를 받는다. n byte 보다 짧은 데이터를 받아도 오류가 나지 않는다.
- **`p.recvn(n)`**
    - n byte 만큼 데이터를 받는다. n byte 보다 짧은 데이터를 받으면 오류가 발생한다.
- **`p.recvline()`**
    - 데이터를 개행 문자를 만날 때까지 받는다.
- **`p.recvuntil(data)`**
    - data가 출력 될 때까지 데이터를 받는다.
- **`p.recvall()`**
    - 프로세스가 종료될 때까지 데이터를 받는다.
- 데이터는 받으면 변수에 저장을 해줘야한다.
    - ex) data = recv(1024)

- **`interactive()`**
    - shell 을 획득했거나, exploit 의 특정 상황에서 직접 입력을 주면서 출력을 확인하고 싶을 때 사용하는 함수
    - 호출하면 터미널로 프로세스에 데이터를 입력할 수 있다. (stdin, stdout 에서 process 로바꿔준다.)

- **`ELF`**
    - exploit 에 사용되는 각종 정보가 삽입되어 있다.
- **`context.log`**
    - debug 의 편의를 돕기 위한 로깅 기능이다.
    - log level 을 context.log_level 변수로 조절할 수 있다.
    - error / debug / info
- **`shellcraft`**
    - shell code 를 쉽게 꺼내 쓸 수 있게 해준다.
- **`asm`**
    - assemble 기능을 실행시켜준다.
    - 아키텍처 정보를 미리 지정해야한다.
